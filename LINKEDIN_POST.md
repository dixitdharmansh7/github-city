## Option 3: Technical Deep Dive

---

I just built a **3D data visualization** that transforms GitHub profiles into a cyberpunk cityscape. Here's the breakdown:

🎯 **Concept**
Each GitHub developer = one 3D building. Metrics determine architecture:
- Commits → Building height (25-180 units)
- Followers → Base width (8-25 units)
- Stars → Window glow intensity
- Primary language → Building color

🏗️ **9 Architecture Types**
Skyscrapers, spires, towers, terraced buildings, campuses, complexes, pyramids, compounds, and monuments - assigned based on developer rank.

🛠️ **Tech Stack**
- React 18 + Vite for blazing fast builds
- Three.js + React Three Fiber for 3D rendering
- React Three Drei for helpers (Stars, OrbitControls)
- Tailwind CSS for glassmorphism UI
- PapaParse for CSV data loading

🎮 **Interactive Features**
- Dual camera: Fly mode (WASD) vs Orbit mode (drag/zoom)
- Search any user by username
- Login to highlight your own building
- Hover tooltips with full stats
- District-based city organization

📦 **Data Pipeline**
Static CSV for MVP → easy to swap for API/DB later without touching the 3D code.

Live: [github-city.vercel.app](https://github-city.vercel.app)  
Repo: github.com/dixitdharmansh7/github-city

Learned a ton about React Three Fiber's performance optimization and Vercel's edge caching!
